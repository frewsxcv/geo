3//! The `geo` crate provides geospatial primitive types and algorithms.
4//!
5//! # Types
6//!
7//! - **[`Coord`]**: A two-dimensional coordinate. All geometry types are composed of [`Coord`]s, though [`Coord`] itself is not a [`Geometry`] type
8//! - **[`Point`]**: A single point represented by one [`Coord`]
9//! - **[`MultiPoint`]**: A collection of [`Point`]s
10//! - **[`Line`]**: A line segment represented by two [`Coord`]s
11//! - **[`LineString`]**: A series of contiguous line segments represented by two or more
12//!   [`Coord`]s
13//! - **[`MultiLineString`]**: A collection of [`LineString`]s
14//! - **[`Polygon`]**: A bounded area represented by one [`LineString`] exterior ring, and zero or
15//!   more [`LineString`] interior rings
16//! - **[`MultiPolygon`]**: A collection of [`Polygon`]s
17//! - **[`Rect`]**: An axis-aligned bounded rectangle represented by minimum and maximum
18//!   [`Coord`]s
19//! - **[`Triangle`]**: A bounded area represented by three [`Coord`] vertices
20//! - **[`GeometryCollection`]**: A collection of [`Geometry`]s
21//! - **[`Geometry`]**: An enumeration of all geometry types, excluding [`Coord`]
22//!
23//! The preceding types are reexported from the [`geo-types`] crate. Consider using that crate
24//! if you only need access to these types and no other `geo` functionality.
25//!
26//! ## Semantics
27//!
28//! The geospatial types provided here aim to adhere to the [OpenGIS Simple feature access][OGC-SFA]
29//! standards. Thus, the types here are inter-operable with other implementations of the standards:
30//! [JTS], [GEOS], etc.
31//!
32//! # Algorithms
33//!
34//! ## Measures
35//!
36//! Algorithms for measures along a line, and how a line is measured.
37//!
38//! ### Metric Spaces
39//!
40//! - **[`Euclidean`]**: The [Euclidean plane] measures distance with the pythagorean formula. Not suitable for lon/lat geometries.
41//! - **[`Haversine`]**: The [Haversine Formula] measures distance on a sphere. Only suitable for lon/lat geometries.
42//! - **[`Geodesic`]**: Geodesic methods based on [Karney (2013)] more accurately reflect the shape of the Earth, but are slower than Haversine. Only suitable for lon/lat geometries.
43//! - **[`Rhumb`]**: [Rhumb line] (a.k.a. loxodrome) measures can be useful for navigation applications where maintaining a constant bearing or direction is important. Only suitable for lon/lat geometries.
44//!
45//! ### Operations on Metric Spaces
46//!
47//! - **[`Distance`]**: Calculate the minimum distance between two geometries.
48//! - **[`Length`]**: Calculate the length of a `Line`, `LineString`, or `MultiLineString`.
49//! - **[`Bearing`]**: Calculate the bearing between two points.
50//!
51//! - **[`Destination`]**: Calculate the destination point from an origin point, given a bearing and a distance.
52//! - **[`InterpolateLine`]**: Interpolate a `Point` along a `Line` or `LineString`.
53//! - **[`InterpolatePoint`]**: Interpolate points along a line.
54//! - **[`Densify`]**: Insert points into a geometry so there is never more than `max_segment_length` between points.
55//!
56//! ### Misc measures
57//!
58//! - **[`HausdorffDistance`]**: Calculate "the maximum of the distances from a point in any of the sets to the nearest point in the other set." (Rote, 1991)
59//! - **[`VincentyDistance`]**: Calculate the minimum geodesic distance between geometries using Vincenty’s formula
60//! - **[`VincentyLength`]**: Calculate the geodesic length of a geometry using Vincenty’s formula
61//! - **[`FrechetDistance`]**: Calculate the similarity between [`LineString`]s using the Fréchet distance
62//!
63//! ## Area
64//!
65//! - **[`Area`]**: Calculate the planar area of a geometry
66//! - **[`ChamberlainDuquetteArea`]**: Calculate the geodesic area of a geometry on a sphere using the algorithm presented in _Some Algorithms for Polygons on a Sphere_ by Chamberlain and Duquette (2007)
67//! - **[`GeodesicArea`]**: Calculate the geodesic area and perimeter of a geometry on an ellipsoid using the algorithm presented in _Algorithms for geodesics_ by Charles Karney (2013)
68//!
69//! ## Boolean Operations
70//!
71//! - **[`BooleanOps`]**: Combine or split (Multi)Polygons using intersection, union, xor, or difference operations
72//! - **[`unary_union`]**: Efficient union of many [`Polygon`] or [`MultiPolygon`]s
73//!
74//! ## Outlier Detection
75//!
76//! - **[`OutlierDetection`]**: Detect outliers in a group of points using [LOF](https://en.wikipedia.org/wiki/Local_outlier_factor)
77//!
78//! ## Simplification
79//!
80//! - **[`Simplify`]**: Simplify a geometry using the Ramer–Douglas–Peucker algorithm
81//! - **[`SimplifyIdx`]**: Calculate a simplified geometry using the Ramer–Douglas–Peucker algorithm, returning coordinate indices
82//! - **[`SimplifyVw`]**: Simplify a geometry using the Visvalingam-Whyatt algorithm
83//! - **[`SimplifyVwPreserve`]**: Simplify a geometry using a topology-preserving variant of the Visvalingam-Whyatt algorithm
84//! - **[`SimplifyVwIdx`]**: Calculate a simplified geometry using the Visvalingam-Whyatt algorithm, returning coordinate indices
85//!
86//! ## Query
87//!
88//! - **[`ClosestPoint`]**: Find the point on a geometry
89//!   closest to a given point
90//! - **[`HaversineClosestPoint`]**: Find the point on a geometry
91//!   closest to a given point on a sphere using spherical coordinates and lines being great arcs
92//! - **[`IsConvex`]**: Calculate the convexity of a [`LineString`]
93//! - **[`LineLocatePoint`]**: Calculate the
94//!   fraction of a line’s total length representing the location of the closest point on the
95//!   line to the given point
96//! - **[`InteriorPoint`]**:
97//!     Calculates a representative point inside a `Geometry`
98//!
99//! ## Topology
100//!
101//! - **[`Contains`]**: Calculate if a geometry contains another
102//!   geometry
103//! - **[`CoordinatePosition`]**: Calculate
104//!   the position of a coordinate relative to a geometry
105//! - **[`HasDimensions`]**: Determine the dimensions of a geometry
106//! - **[`Intersects`]**: Calculate if a geometry intersects
107//!   another geometry
108//! - **[`line_intersection`]**: Calculates the
109//!   intersection, if any, between two lines
110//! - **[`Relate`]**: Topologically relate two geometries based on
111//!   [DE-9IM](https://en.wikipedia.org/wiki/DE-9IM) semantics
112//! - **[`Within`]**: Calculate if a geometry lies completely within another geometry
113//!
114//! ## Triangulation
115//!
116//! - **[`TriangulateEarcut`](triangulate_earcut)**: Triangulate polygons using the earcut algorithm. Requires the `earcutr` feature, which is enabled by default
117//! - **[`TriangulateDelaunay`](triangulate_delaunay)**: Produce constrained or unconstrained Delaunay triangulations of polygons. Requires the `spade` feature, which is enabled by default
118//! ## Winding
119//!
120//! - **[`Orient`]**: Apply a specified winding [`Direction`](orient::Direction) to a [`Polygon`]’s interior and exterior rings
121//! - **[`Winding`]**: Calculate and manipulate the [`WindingOrder`](winding_order::WindingOrder) of a [`LineString`]
122//!
123//! ## Iteration
124//!
125//! - **[`CoordsIter`]**: Iterate over the coordinates of a geometry
126//! - **[`MapCoords`]**: Map a function over all the coordinates
127//!   in a geometry, returning a new geometry
128//! - **[`MapCoordsInPlace`]**: Map a function over all the
129//!   coordinates in a geometry in-place
130//! - **[`LinesIter`]**: Iterate over lines of a geometry
131//!
132//! ## Boundary
133//!
134//! - **[`BoundingRect`]**: Calculate the axis-aligned
135//!   bounding rectangle of a geometry
136//! - **[`MinimumRotatedRect`]**: Calculate the
137//!   minimum bounding box of a geometry
138//! - **[`ConcaveHull`]**: Calculate the concave hull of a
139//!   geometry
140//! - **[`ConvexHull`]**: Calculate the convex hull of a
141//!   geometry
142//! - **[`Extremes`]**: Calculate the extreme coordinates and
143//!   indices of a geometry
144//!
145//! ## Affine transformations
146//!
147//! - **[`Rotate`]**: Rotate a geometry around its centroid
148//! - **[`Scale`]**: Scale a geometry up or down by a factor
149//! - **[`Skew`]**: Skew a geometry by shearing angles along the `x` and `y` dimension
150//! - **[`Translate`]**: Translate a geometry along its axis
151//! - **[`AffineOps`]**: generalised composable affine operations
152//!
153//! ## Conversion
154//!
155//! - **[`Convert`]**: Convert (infalliby) the numeric type of a geometry’s coordinate value
156//! - **[`TryConvert`]**: Convert (falliby) the numeric type of a geometry’s coordinate value
157//! - **[`ToDegrees`]**: Radians to degrees coordinate transforms for a given geometry
158//! - **[`ToRadians`]**: Degrees to radians coordinate transforms for a given geometry
159//!
160//! ## Miscellaneous
161//!
162//! - **[`Centroid`]**: Calculate the centroid of a geometry
163//! - **[`ChaikinSmoothing`]**: Smoothen `LineString`, `Polygon`, `MultiLineString` and `MultiPolygon` using Chaikin's algorithm
164//! - **[`proj`]**: Project geometries with the `proj` crate (requires the `use-proj` feature)
165//! - **[`LineStringSegmentize`]**: Segment a LineString into `n` segments
166//! - **[`LineStringSegmentizeHaversine`]**: Segment a LineString using Haversine distance
167//! - **[`Transform`]**: Transform a geometry using Proj
168//! - **[`RemoveRepeatedPoints`]**: Remove repeated points from a geometry
169//! - **[`Validation`]**: Checks if the geometry is well formed. Some algorithms may not work correctly with invalid geometries
170//!
171//! # Spatial Indexing
172//!
173//! `geo` geometries ([`Point`], [`Line`], [`LineString`], [`Polygon`], [`MultiPolygon`]) can be used with the [rstar](https://docs.rs/rstar/0.12.0/rstar/struct.RTree.html#usage)
174//! R*-tree crate for fast distance and nearest-neighbour queries. Multi- geometries can be added to the tree by iterating over
175//! their members and adding them. Note in particular the availability of the [`bulk_load`](https://docs.rs/rstar/0.12.0/rstar/struct.RTree.html#method.bulk_load)
176//! method and [`GeomWithData`](https://docs.rs/rstar/0.12.0/rstar/primitives/struct.GeomWithData.html) struct.
177//!
178//! # Features
179//!
180//! The following optional [Cargo features] are available:
181//!
182//! - `earcutr`:
183//!     - Enables the `earcutr` crate, which provides triangulation of polygons using the earcut algorithm
184//!     - ☑ Enabled by default
185//! - `proj-network`:
186//!     - Enables [network grid] support for the [`proj` crate]
187//!     - After enabling this feature, [further configuration][proj crate file download] is required to use the network grid.
188//!     - ☐ Disabled by default
189//! - `use-proj`:
190//!     - Enables coordinate conversion and transformation of `Point` geometries using the [`proj` crate]
191//!     - ☐ Disabled by default
192//! - `use-serde`:
193//!     - Allows geometry types to be serialized and deserialized with [Serde]
194//!     - ☐ Disabled by default
195//! - `multithreading`:
196//!     - Enables multithreading support (via Rayon), and activates the `multithreading` flag
197//!       in `geo-types`, enabling multi-threaded iteration over `Multi*` geometries
198//!     - ☑ Enabled by default

287/// A common numeric trait used for geo algorithms
288///
289/// Different numeric types have different tradeoffs. `geo` strives to utilize generics to allow
290/// users to choose their numeric types. If you are writing a function which you'd like to be
291/// generic over all the numeric types supported by geo, you probably want to constrain
292/// your function input to `GeoFloat`. For methods which work for integers, and not just floating
293/// point, see [`GeoNum`].
294///
295/// # Examples
296///
297/// ```
298/// use geo::{GeoFloat, MultiPolygon, Polygon, Point};
299///
300/// // An admittedly silly method implementation, but the signature shows how to use the GeoFloat trait
301/// fn farthest_from<'a, T: GeoFloat>(point: &Point<T>, polygons: &'a MultiPolygon<T>) -> Option<&'a Polygon<T>> {
302///     polygons.iter().fold(None, |accum, next| {
303///         match accum {
304///             None => Some(next),
305///             Some(farthest) => {
306///                 use geo::{euclidean_distance::EuclideanDistance};
307///                 if next.euclidean_distance(point) > farthest.euclidean_distance(point) {
308///                     Some(next)
309///                 } else {
310///                     Some(farthest)
311///                 }
312///             }
313///         }
314///     })
315/// }
316/// ```
317pub trait GeoFloat:
318    GeoNum + num_traits::Float + num_traits::Signed + num_traits::Bounded + float_next_after::NextAfter
319{
320}
321impl<T> GeoFloat for T where
322    T: GeoNum
323        + num_traits::Float
324        + num_traits::Signed
325        + num_traits::Bounded
326        + float_next_after::NextAfter
327{
328}

330/// A trait for methods which work for both integers **and** floating point
331pub trait GeoNum: CoordNum {
332    type Ker: Kernel<Self>;
333
334    /// Return the ordering between self and other.
335    ///
336    /// For integers, this should behave just like [`Ord`].
337    ///
338    /// For floating point numbers, unlike the standard partial comparison between floating point numbers, this comparison
339    /// always produces an ordering.
340    ///
341    /// See [f64::total_cmp](https://doc.rust-lang.org/src/core/num/f64.rs.html#1432) for details.
342    fn total_cmp(&self, other: &Self) -> Ordering;
343}
344
345macro_rules! impl_geo_num_for_float {
346    ($t: ident) => {
347        impl GeoNum for $t {
348            type Ker = RobustKernel;
349            fn total_cmp(&self, other: &Self) -> Ordering {
350                self.total_cmp(other)
351            }
352        }
353    };
354}
355macro_rules! impl_geo_num_for_int {
356    ($t: ident) => {
357        impl GeoNum for $t {
358            type Ker = SimpleKernel;
359            fn total_cmp(&self, other: &Self) -> Ordering {
360                self.cmp(other)
361            }
362        }
363    };
364}

366// This is the list of primitives that we support.
367impl_geo_num_for_float!(f32);
368impl_geo_num_for_float!(f64);
369impl_geo_num_for_int!(i16);
370impl_geo_num_for_int!(i32);
371impl_geo_num_for_int!(i64);
372impl_geo_num_for_int!(i128);
373impl_geo_num_for_int!(isize);

380    fn total_ord_float() {
381        assert_eq!(GeoNum::total_cmp(&3.0f64, &2.0f64), Ordering::Greater);
382        assert_eq!(GeoNum::total_cmp(&2.0f64, &2.0f64), Ordering::Equal);
383        assert_eq!(GeoNum::total_cmp(&1.0f64, &2.0f64), Ordering::Less);
384        assert_eq!(GeoNum::total_cmp(&1.0f64, &f64::NAN), Ordering::Less);
385        assert_eq!(GeoNum::total_cmp(&f64::NAN, &f64::NAN), Ordering::Equal);
386        assert_eq!(GeoNum::total_cmp(&f64::INFINITY, &f64::NAN), Ordering::Less);
387    }
